This work is part of a larger effort on producing NIEM-based SOI-provided data types. The effort is driven by a call for the adoption of a [ NIEM-based ] approach to the implemention of standards-based XML data exchanges. Therefore, the work implements the NIEM data component framework, which includes selected community described data components, wherever applicable. In addition, the work follows NIEM guidelines and conformance rules.

This work also strives to create a product that software developers can deploy, exercise, and reuse. Beyond creating static XML Schema documents, this work includes the following:

  • Git repository for maximum portability
  • Ant and Maven-driven test, build, and packaging automation
  • XSL 3.0 XML and JSON data format transformations
  • XSL Schematron build and application rule validation

Directive

The memorandum "Adoption of the National Information Exchange Model within the Department of Defense" (March 28, 2013) calls for the adoption of the NIEM-based approach for implementing standards-based XML data exchanges.

Adoption

SOI-provided XML data types shall be migrated over time in accordance to the NIEM-based approach. NIEM-based data types may be deployed in parallel with non-NIEM-based data types. Parallel support of both data types should maximize the capatibilities of end-points that only support non-NIEM-based data types. Non-NIEM-based data types shall be sunset when all relevant end-points support NIEM-based data types.

New SOI XML data types shall be developed in accordance to the NIEM-based approach.

Development

The development of a SOI-provided NIEM-based data type shall include the following:

  • [ Identify and swap ] data components with NIEM data components, where possible
  • [ Encode ] XML Schema in accordance to [ NIEM guidelines ]
  • Map data components for a non-NIEM-based data type to a NIEM-based data type, and the converse
  • Develop a capability to validate application rules
  • Develop a capability to transform a non-NIEM-based data type to a NIEM-based data type, and the converse
  • Automate the validation of data structures and application rules for XML instances
  • Automate the testing and build process that assembles the [ NIEM-based data type package ]
  • Automate the distribution of changes to managed repositories, to include Software Forge.mil at project [ USMC - MAGTF C2 ] SVN repo [ SoiMessaging ].

Implementation

The implementation of a SOI-provided NIEM-based data type, first and foremost, is dependent on having an end-point that will support a NIEM-based data type. An end-point that supports a NIEM-based data type may also support a non-NIEM-based data type. The NIEM-based data type package includes validation and transformation capabilities the end-point may implement.

Maintenance

As with other software components, a data type and its packaged components should evolve to improve their usability and relevancy, both for information systems and their builders. Towards this premise, SOI-provided XML data types and their packaged components shall be maintained.
